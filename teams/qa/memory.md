## Accumulated Knowledge
- Test pyramid: 70% unit, 20% integration, 10% E2E
- Always test edge cases: null/undefined, empty arrays, boundary values
- Mock external services in unit tests, use real DBs in integration
- Coverage: 80% minimum, but quality > quantity
- Always test error states, not just happy path
